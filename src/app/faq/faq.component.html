Explain the ineffable

Q: Okay, seriously, what's going on here?

A: 0xmons is an experimental NFT project with completely unique, animated pixel monsters that are all algorithmically generated (but human-curated).


Q: Where is the contract? Is it audited?

A: You can find it here. It is not audited.


Q: Is there a GitHub repo?

A: Yes, check here.


Q: Okay, so how do I get a monster?

A: You can burn a gem (which you can get for staking your yfBeta at yfLambda) that has been staked for the minimum stake amount for the minimum stake time for a monster. Or you can purchase one directly.


Q: What happens to my gem if I redeem a monster?

A: It goes to the 0 address and is burned.


Q: Can I still unstake my YFB from yfLambda if I burn my gem?

A: Yes.


Q: How many monsters can I own?

A: As many as you want.


Q: What's next?

A: Battling. More monsters.


Q: How does spawning work?

A: You pick two available monsters and use them to create a new monster. You still get to keep your original monsters. After spawning, both the two parents and the new monster will be locked from spawning for a set amount of time. You can see this on the Bestiary page where it shows the unblock block for each monster.


Q: Okay, but how do 0xmons actually work behind the scenes?

Open long text box
Every monster has the following attributes:
  • miner
    • This is the address that originally created the monster.
  • unlockBlock
    • This is the block where the monster can be used to spawn new ones again.
  • parent1
    • This is the ID of the first parent. It is 0 if there is no parent.
  • parent2
    • This is the ID of the second parent. It is 0 if there is no parent.
  • gen
    • This is the number of generations in this monster's family tree.
  • amount
    • This is the amount of YFB that was in the gem that was burned to create this monster. If the monster was bought, this value is 0. If the monster was spawned, it is the sum of its parents' amounts.
  • duration
    • The is the amount of time that underlying YFB was staked before the gem was burnt to create this monster. If the monster was bought, this value is 0. If the monster was spawned, it is the sum of its parents' durations.
  • powerBits
    • This is the blockhash of the previous number, then summed up with the monster's duration and amount.
  • series
    • This is the series of the monster, which goes up as new monsters are added.

Non-spawned monsters are generated off-chain and then uploaded. When you receive a new monster, it gets the image from the server. This can be done ahead of time. Spawned monsters require additional computation because the model has to run with the specific inputs that created the two parent monsters.


Q: What can the owner of the contract do?

A: Transfer tokens out and change the parameters like minimum stake time, stake amount, and price to buy a monster.
